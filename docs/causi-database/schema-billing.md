# Schema `billing` — SaaS Business

O schema `billing` gerencia planos, assinaturas, addons e pagamentos da plataforma Causi.

> **Fonte**: Os arquivos SQL de cada tabela estão em [`supabase/schemas/billing/tables/`](../../supabase/schemas/billing/tables/).
> Para valores reais dos planos, veja [plans-and-features.md](../references/plans-and-features.md).

---

## Visão Geral

O schema possui **7 tabelas** distribuídas em 3 grupos:

1. **Catálogo** — Definição de ofertas (`plans`, `plan_periods`, `addons`, `addon_prices`)
2. **Assinaturas** — Contratos ativos (`subscriptions`, `subscription_addons`)
3. **Financeiro** — Histórico de cobranças (`payments`)

Todas as tabelas usam `bigint GENERATED BY DEFAULT AS IDENTITY` como PK, e incluem as colunas `created_at` e `updated_at` do tipo `timestamptz NOT NULL DEFAULT now()`.

---

## 1. Catálogo

### `plans`

Definição dos planos disponíveis na plataforma.

| Coluna | Tipo | Descrição |
|--------|------|-----------|
| `id` | `bigint` (identity) | PK |
| `created_at` | `timestamptz NOT NULL` | Timestamp de criação (default: `now()`) |
| `updated_at` | `timestamptz NOT NULL` | Timestamp de atualização (default: `now()`) |
| `name` | `text NOT NULL` | Nome do plano (ex: "Essencial") |
| `slug` | `text NOT NULL UNIQUE` | Identificador (ex: `essential`) |
| `description` | `text` | Descrição |
| `base_price` | `numeric NOT NULL` | Preço base mensal em BRL (ex: `127`) — **não** em centavos |
| `base_limits` | `jsonb` | Limites base: `{ max_users, max_channels, max_contacts, max_pipelines }` |
| `features` | `jsonb` | Features habilitadas: `{ tags: true, deals: true, agents: false, classroom: true, ... }` |
| `is_active` | `boolean NOT NULL` | Plano visível para clientes (default: `false`) |
| `order` | `real NOT NULL` | Posição de exibição |
| `trial_days` | `smallint NOT NULL` | Dias de trial (default: `0`) |
| `tier_level` | `smallint NOT NULL` | Nível para lógica de upgrade/downgrade (default: `0`) |

### `plan_periods`

Ciclos de faturamento com descontos progressivos.

| Coluna | Tipo | Descrição |
|--------|------|-----------|
| `id` | `bigint` (identity) | PK |
| `created_at` | `timestamptz NOT NULL` | Timestamp de criação (default: `now()`) |
| `updated_at` | `timestamptz NOT NULL` | Timestamp de atualização (default: `now()`) |
| `name` | `text NOT NULL` | Nome (ex: "Semestral") |
| `slug` | `text NOT NULL UNIQUE` | Identificador (ex: `semiannually`) |
| `months` | `smallint NOT NULL` | Duração em meses |
| `discount_percentage` | `numeric NOT NULL` | Percentual de desconto (default: `0`) |
| `is_active` | `boolean NOT NULL` | Período disponível (default: `false`) |

### `addons`

Complementos que expandem limites sem trocar de plano.

| Coluna | Tipo | Descrição |
|--------|------|-----------|
| `id` | `bigint` (identity) | PK |
| `created_at` | `timestamptz NOT NULL` | Timestamp de criação (default: `now()`) |
| `updated_at` | `timestamptz NOT NULL` | Timestamp de atualização (default: `now()`) |
| `name` | `text NOT NULL` | Nome (ex: "Usuário Adicional") |
| `slug` | `text NOT NULL UNIQUE` | Identificador (ex: `additional-users`) |
| `description` | `text` | Descrição |
| `type` | `text NOT NULL` | Tipo de limite expandido: `'users'`, `'channels'`, `'contacts'` |
| `unit_quantity` | `smallint NOT NULL` | Quantidade por unidade (default: `1`) |
| `max_quantity` | `smallint` | Máximo de unidades por assinatura |
| `is_active` | `boolean NOT NULL` | Addon disponível (default: `false`) |

### `addon_prices`

Preços dos addons diferenciados por plano.

| Coluna | Tipo | Descrição |
|--------|------|-----------|
| `id` | `bigint` (identity) | PK |
| `created_at` | `timestamptz NOT NULL` | Timestamp de criação (default: `now()`) |
| `updated_at` | `timestamptz NOT NULL` | Timestamp de atualização (default: `now()`) |
| `addon_id` | `bigint NOT NULL` | FK → `addons(id)` |
| `plan_id` | `bigint` | FK → `plans(id)` — preço específico por plano |
| `price_per_unit` | `numeric NOT NULL` | Preço por unidade em BRL |
| `is_active` | `boolean NOT NULL` | Preço ativo (default: `false`) |

## 2. Assinaturas

### `subscriptions`

Contrato ativo entre uma `account` e o Causi. Cada conta pode ter múltiplos registros históricos (ao trocar de plano, a antiga é cancelada e uma nova é criada). Só pode haver uma assinatura ativa por conta.

| Coluna | Tipo | Descrição |
|--------|------|-----------|
| `id` | `bigint` (identity) | PK |
| `created_at` | `timestamptz NOT NULL` | Timestamp de criação (default: `now()`) |
| `updated_at` | `timestamptz NOT NULL` | Timestamp de atualização (default: `now()`) |
| `account_id` | `bigint NOT NULL` | FK → `public.accounts(id)` |
| `plan_id` | `bigint NOT NULL` | FK → `plans(id)` |
| `plan_period_id` | `bigint` | FK → `plan_periods(id)` |
| `period_months` | `smallint` | Duração em meses |
| `start_date` | `timestamptz` | Início da vigência |
| `end_date` | `timestamptz` | Fim da vigência |
| `trial_start_date` | `timestamptz` | Início do trial |
| `trial_end_date` | `timestamptz` | Fim do trial |
| `status` | `text NOT NULL` | `'pending'`, `'trial'`, `'active'`, `'cancelled'` (default: `'pending'`) |
| `base_price` | `numeric NOT NULL` | Preço base no momento da contratação (default: `0`) |
| `base_price_with_discount` | `numeric NOT NULL` | Preço base com desconto de período (default: `0`) |
| `addons_total` | `numeric NOT NULL` | Preço total de addons (default: `0`) |
| `total_price` | `numeric NOT NULL` | Preço total consolidado: base_price_with_discount + addons_total (default: `0`) |
| `total_limits` | `jsonb` | Limites consolidados: base_limits + addons |
| `payment_status` | `text` | Status de pagamento (valores livres) |
| `payment_method` | `text` | Método de pagamento |
| `next_due_date` | `timestamptz` | Próximo vencimento |
| `cancelled_at` | `timestamptz` | Data de cancelamento |
| `cancel_requested_at` | `timestamptz` | Data de solicitação de cancelamento |
| `cancel_reason` | `text` | Motivo do cancelamento (pelo usuário) |
| `system_cancel_reason` | `text` | Motivo do cancelamento (pelo sistema) |
| `is_granted` | `boolean NOT NULL` | Cortesia/acesso administrativo (default: `false`) |
| `granted_by_user_id` | `uuid` | FK → `public.users(id)` — quem concedeu |
| `management_type` | `text` | Tipo de gestão: `'platform'` (default) |

### `subscription_addons`

Addons vinculados a uma assinatura específica.

| Coluna | Tipo | Descrição |
|--------|------|-----------|
| `id` | `bigint` (identity) | PK |
| `created_at` | `timestamptz NOT NULL` | Timestamp de criação (default: `now()`) |
| `updated_at` | `timestamptz NOT NULL` | Timestamp de atualização (default: `now()`) |
| `subscription_id` | `bigint NOT NULL` | FK → `subscriptions(id)` |
| `addon_id` | `bigint NOT NULL` | FK → `addons(id)` — UNIQUE com `subscription_id` |
| `quantity` | `smallint NOT NULL` | Quantidade contratada (default: `1`) |
| `unit_price` | `numeric NOT NULL` | Preço unitário no momento da contratação |
| `total_price` | `numeric NOT NULL` | Preço total (unit_price × quantity) |
| `discount_percentage` | `numeric NOT NULL` | Desconto aplicado (default: `0`) |
| `unit_quantity` | `smallint` | Quantidade por unidade do addon (default: `1`) |

## Relacionamentos

```
public.accounts ─────── subscriptions (account_id)

public.users ─────────── subscriptions (granted_by_user_id)

plans ────────────────── addon_prices (plan_id)
                     └── subscriptions (plan_id)

plan_periods ──────────── subscriptions (plan_period_id)

addons ───────────────── addon_prices (addon_id)
                     └── subscription_addons (addon_id)

subscriptions ──────────── subscription_addons (subscription_id)
                       └── payments (subscription_id)
```

> A notação `parent ─── child (fk_column)` indica que a coluna `fk_column` está na tabela filho e aponta para o pai.

## Segurança & RLS

- Tabelas de catálogo (`plans`, `plan_periods`, `addons`, `addon_prices`) permitem `SELECT` para **qualquer usuário autenticado** (`USING (true)` — sem filtro por `is_active`); `INSERT/UPDATE/DELETE` restrito a `super_admin`.
- `subscriptions`: SELECT restrito a membros da conta (`is_user_in_account_or_shared`). INSERT permitido para o próprio usuário apenas nos casos: plano gratuito (`plan_id = 1`) ou plano educacional (`plan_id = 8` com código secreto `EDU_CLASSROOM_2026` nos metadados do usuário). Demais operações restritas a `super_admin`.
- `subscription_addons`: **sem política SELECT para usuários comuns** — apenas `super_admin` tem acesso direto.
- `payments`: SELECT permitido para membros da conta via subquery em `subscriptions.account_id`; `INSERT/UPDATE/DELETE` restrito a `super_admin`.

> Detalhes completos em [rls.md](./rls.md) e [`supabase/schemas/billing/rls/`](../../supabase/schemas/billing/rls/).
